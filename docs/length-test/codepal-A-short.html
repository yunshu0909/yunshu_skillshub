CodePal

AI 写代码，但谁来帮你管 AI？
📋 本次配置 · 受众：PM / 产品经理 · 终点：概念理解（看完知道 CodePal 是什么、解决什么、有哪些核心模块）· 长度：短报告 ≤ 2K 字 / 5 分钟 · 风格：故事化阅读（暖色书卷）
一句话：CodePal 不和 Claude Code、Codex、Cursor 抢"写代码"这件事，它做的是这些工具之间的连接层——把 Skills 同步、账户切换、用量监控、对话回顾这些"AI 工具不会替你做、但你每天都在做"的活儿，一次性收编。

一一个 PM 没意识到的日常摩擦

想象你是一个重度 AI 编程用户。早上你用 Claude Code 写需求池脚本，中午切到 Codex 改一段算法，下午用 Cursor 调前端，晚上又回到 Claude Code 写复盘。

四个工具，看起来都很顺手。但有些事没人替你做：

几十个自定义 Skills，要在三个工具之间手动同步——改一处要到处改。Codex Plus 有 5 小时额度墙，触顶就得 Logout、换邮箱、重登、二次验证，1-2 分钟还得记密码。这个月 Token 花了多少钱？哪个模型最费？没人告诉你。上次和 AI 聊过的那条对话在哪？翻 Session 目录。新项目想要标准的 CLAUDE.md？每次手动抄。

问题在于：没有任何一个 AI 工具会帮你做"跨工具"的事。Claude Code 不管 Codex，Codex 不管 Cursor。每个工具都只对自己负责，缝隙里的活儿全是你自己的。

二转折：把"缝隙"做成产品

CodePal 的定位很克制——它不是第五个 AI 编程工具。它是一个躲在所有 AI 工具背后的桌面助手，按四类日常摩擦拆成四个模块：

🔋 账户与用量

Codex 多账户秒切（<200ms，不重登）、凭证后台保活、Claude 会员 5h/7d 额度状态栏、按模型/项目的 Token 花费统计。触顶前你就知道。

📚 文档

对话回顾——浏览所有 Claude Code 历史 Session，全文搜索 + 一键 resume；多目录 Markdown 浏览器，把散落在各处的文档拢成一棵树。找得到、也接得回去。

🛠 技能中心

Skills 一次扫描，三个工具一键同步；MCP 服务可视化启停。写一次，到处用，改一处全局生效。

⚙️ 工具设置

Claude Code 启动权限模式四档切换、新项目模板一键生成（CLAUDE.md / .gitignore / 记忆系统）、网络与 API 连通性诊断。把重复劳动模板化。

三启示：痛点驱动的飞轮

CodePal 没有路线图，也不提前规划功能列表。作者自述只有一句话：用 → 痛 → 解决 → 沉淀。

这意味着每一个模块都不是凭空想象的需求，而是作者本人在日常重度使用 Claude Code / Codex / Cursor 的过程中，被某个具体动作磨痒了，才长出来的功能。Codex 多账户秒切是因为他一天切 N 次受不了；对话回顾是因为他真的找不到上次那段聊天；Skills 同步是因为他维护着几十个 skill 不想三处复制粘贴。

对 PM 来说，这是一个有趣的样本：当用户和开发者是同一个人时，需求池天然没有水分。代价是产品不会去抢"通用 AI 编程助手"的故事，受众清晰但收窄——只服务那批每天在多个 AI 工具之间穿梭的重度用户。

读完记一句话：CodePal = AI 编程工具的"幕后管家"，专管账户、文档、技能、工具配置这四类跨工具的杂活。

什么时候它不适合你：如果你只用一个 AI 工具、不在意 Token 花费、也没攒过自定义 Skills——那 CodePal 解决的痛你还没遇到，等遇到再来不迟。
