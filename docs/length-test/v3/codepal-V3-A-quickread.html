CodePal —— AI 编程的"幕后助手"

写代码归 AI，写代码之外的杂事归它。
一个真实场景：上午 10 点，你在 Claude Code 里写功能，额度突然飘红；切到 Codex，账号 token 过期，重登 + 二次验证花了 90 秒；想起一条上周和 AI 讨论过的方案，翻 Session 目录翻了 5 分钟没找到；同时 Cursor 那边的 skill 还没同步。
—— 写代码本身只占了 30%，剩下都是工具之间的摩擦。
TL;DR

CodePal 不替你写代码，而是把 Claude Code / Codex / Cursor / Trae 四个工具之间的"接缝"接起来——账户秒切、Skills 同步、用量监控、对话回溯、模板生成，全在一个 macOS 原生壳子里。一句话：AI 工具各自专心写代码，连接层归 CodePal。

CodePal 站在四个 AI 编程工具中间，做"它们之间"的事

它具体管的 5 件事

  1. 账户与用量　Codex 多账户秒切（<200ms，凭证后台保活）+ Claude 会员 5h/7d 额度状态栏 + Token 花了多少钱按模型/项目可视化。
  2. Skills 中央仓库　从任意工具扫描导入，一键推送到所有工具，改一处全局生效，不用再手动复制。
  3. 对话回顾　全文搜索历史 Session，一键复制 claude --resume 命令或直接新终端启动，接着上次聊。
  4. MCP & 启动模式　按工具管理 MCP 启停、Claude Code 4 种权限模式一键切换，不用手编 JSON。
  5. 新建项目模板　选好模板一键生成 CLAUDE.md / .gitignore / 记忆系统，省掉每次从零抄。
"AI 写代码，CodePal 管周边。"
—— 拿这一句去给同事种草，比解释 5 件事更快。
