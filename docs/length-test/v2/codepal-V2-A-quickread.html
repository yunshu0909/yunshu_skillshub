TL;DR

AI 编程工具（Claude Code / Codex / Cursor / Trae）只管写代码，不管彼此之间的事。CodePal 就是那一层"幕后助手"——把 Skills 同步、账户切换、用量监控、对话回顾这些跨工具的杂活从你手里接走。

用 AI 写代码很爽，用得越多越发现瓶颈不在代码本身：Skill 改一处要在三个工具里同步、Codex 额度一满就得 Logout 重登 + 二次验证、月底想看花了多少 token 没人告诉你、上周那段精彩对话翻不到 Session 目录。

这些事 Claude Code 不管 Codex，Codex 也不管 Cursor——没有一个 AI 工具会替你做"跨工具的事"。CodePal 的产品定位就一句话：它是 AI 编程工具之间的连接层。它不替任何工具写代码，只把它们之间的摩擦磨平，让"用 AI 编程"这件事从单点工具升级成一套顺手的工作流。

这也决定了它的产品哲学：不提前规划功能，而是按"用 → 痛 → 解决 → 沉淀"的飞轮迭代——作者自用驱动，每一个模块都来自真实的日常摩擦点，而不是想象出来的需求。

不要把它当成：另一个 AI 编程工具，也不要指望它替你写 Skill 内容——它不写代码、不做推理、不生产 Skill，只做"工具周边的调度和管理"。如果你只用一个 AI 工具且从不切账号、不写 Skill，CodePal 给你的价值会很有限。
"AI 写代码，CodePal 写代码之外的一切。"
适用判断：① 同时用 ≥ 2 个 AI 编程工具；② 有多个 Codex 账户在轮换；③ 在意 Token 花了多少钱 / Skill 想沉淀复用 —— 命中任一项即可考虑。当前仅支持 macOS Apple Silicon，从 GitHub Releases 下载 dmg 安装即可上手。
