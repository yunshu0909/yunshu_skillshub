# 案例：Claude Skills 文章写作过程

> 这是一个完整的从思维挖掘到成稿的案例，记录了整个写作过程

---

## 背景

用户想写一篇关于 Claude Skills 的文章，但不知道写什么。通过对话的方式，帮助用户整理思路、确定选题、完成写作。

---

## 第一阶段：思维挖掘

**过程：**
- 用户零散分享了对 Skills 的各种理解
- 通过不断提问，收集了 20 个洞察点

**收集到的主要洞察：**
1. Skills 是 Prompt 的模块化升级
2. 渐进式演化，而非顶层设计
3. 超大号的 SOP + 知识库
4. 框架是容器，经验才是内核
5. 自我迭代的动态知识库
6. Skills 的边界——不是万能的
7. 完整的演化链条（基础模型→Function Call→MCP→Agent→Skills）
8. Skills 需要定制，通用性有限
9. Skills vs GPTs 的本质区别
10. 做 Skills 没有门槛
...（共 20 个洞察）

---

## 第二阶段：选题确定

**遇到的问题：**
- 尝试了 4 个选题角度，用户都觉得"没什么意思"
- 意识到这些角度太"内视角"，讲的是 Skills 本身，而不是读者关心的问题

**转折点：**
- 问用户："如果只能告诉别人一句话，你会说什么？"
- 用户回答："Skills 是工作流 + 知识库，让 Agent 更全能"

**追问后确定的核心观点：**
> Skills 不是大号提示词，是「动态工作流 + 知识库」，由 Agent 能力驱动

**关键区分：**
- 传统工作流：写死的 ABCDE
- Skills：Agent 根据情况智能调度

---

## 第三阶段：观点验证

**联网搜索验证：**
- 搜索了市场上对 Skills 的理解
- 对比用户的观点和官方定义

**验证结果：**
- 用户的理解没有问题
- 和官方的"渐进式加载"概念一致
- 补充了"Skills 本身没有 MCP 调用能力，是通过 Agent 调用"

---

## 第四阶段：写作辅助

**用户写作方式：**
- 用对话的形式展开（写作助手 Skills 提问，用户回答）
- 边写边发过来检查

**辅助内容：**
1. 逻辑检查：确认和之前讨论的洞察一致
2. 润色建议：找出可以更顺的表达
3. 金句标注：
   - "Skills 让 Agent 从实习生变成资深老员工"
   - "没有 Skills 的 Agent，是个聪明但容易忘事的实习生"

---

## 第五阶段：最终审核

**检查结果：**
- [x] 主题清晰：Skills 的核心价值 + 和提示词的区别
- [x] 逻辑通顺：层层递进，无矛盾
- [x] 事实正确：已联网验证
- [x] 有金句：实习生→老员工
- [x] 结尾有力：建议稍作调整

**最终评价：** 可以发

## 经验总结

1. **"一句话"问题很有效** — 帮用户从一堆洞察中找到核心
2. **追问"为什么重要"** — 帮用户把定义变成观点
3. **联网验证很必要** — 给用户信心，确认逻辑没问题
4. **对话形式写作** — 用户更容易输出，比直接写更自然
