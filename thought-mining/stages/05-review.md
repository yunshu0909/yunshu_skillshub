# 第五阶段：最终审核

**目标：** 发布前的最后检查

---

## 检查清单

逐项检查并告诉用户结果：

- [ ] **主题清晰吗？**
  - 读者能一句话说出这篇文章讲什么吗？
  - 如果不能，说明主题不够清晰

- [ ] **逻辑通顺吗？**
  - 有没有跳跃或矛盾的地方？
  - 段落之间的衔接自然吗？

- [ ] **事实正确吗？**
  - 涉及的技术概念、产品功能有没有错？
  - 已在第三阶段验证过，再确认一次

- [ ] **有没有金句？**
  - 有没有能被记住/传播的句子？
  - 如果没有，建议用户加一个

- [ ] **结尾有力吗？**
  - 结尾是不是太弱了？
  - 好的结尾应该让人有回味或行动的冲动

---

## 输出格式

```
## 最终审核结果

✓ 主题清晰：{评价}
✓ 逻辑通顺：{评价}
✓ 事实正确：{评价}
✓ 金句：{列出金句}
✓ 结尾：{评价}

**综合评价：** 可以发 / 建议再改

**如需修改：**
- {具体位置和建议}
```

---

## 本阶段结束标志

- 用户确认"可以发了"

---

## 结束后

告诉用户：「审核完成，文章可以发布了。祝发布顺利！如果后续有读者反馈，可以告诉我，我们继续迭代这个 Skill。」
