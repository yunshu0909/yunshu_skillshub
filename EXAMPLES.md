## 🎨 配图助手 (Image Assistant)

### 场景：为一篇技术文章配图

```
用户: 我写了一篇介绍 AI 工具选型的文章，需要配几张图
助手: [自动触发配图助手流程]
助手: 好的！让我先了解一下需求：
     1. 这篇文章主要用在哪里？（PPT、公众号、博客等）
     2. 谁来看？（技术人员、产品经理、普通用户？）
     3. 你希望图上是更简洁少字，还是信息密度高一些？

用户: 用在公众号文章里，面向产品经理和技术人员，希望简洁易读
助手: 明白了！让我看看文章内容...

     【第2步：配图规划】
     根据你的内容，我建议做 3 张图：
     1. 封面图：AI 工具选型的整体框架
     2. 对比图：开源 vs 商业方案的对比
     3. 决策流程图：如何选择合适的工具

     这样可以吗？

用户: 可以！
助手: 【第3步：文案定稿】
     第1张-封面图的文案：
     - 大标题：AI 工具选型指南
     - 副标题：场景→需求→选型
     - 3个关键词：性能、成本、生态

     确认这些文案吗？

[... 继续逐张确认文案 ...]

助手: 【第4步：提示词输出】
     第1张图的生图提示词：

\`\`\`
16:9横版信息图。奶油纸底+彩铅水彩手绘风格。
中心大标题【AI工具选型指南】，下方流程：场景分析→需求定义→工具选型。
底部3个图标+关键词：性能⚡、成本💰、生态🌱。
少字高可读，干净趣味。
\`\`\`

     可以直接复制提示词去生成图片了！需要调整吗？
```

**预期输出**:
- 图片清单（几张图/每张目的）
- 每张图的文案定稿（Copy Spec）
- 可复制的生图提示词
- （可选）批量生成的图片

## 🧠 思维挖掘助手 (Thought Mining)

### 场景：想写一篇关于产品设计的文章

```
用户: 我想写一篇关于产品设计的文章
助手: [自动触发思维挖掘流程]
助手: 好的！我们进入思维挖掘阶段。让我们先聊聊，你对产品设计有什么想法？
     随便说，想到什么说什么，我来帮你记录。

用户: 我觉得很多产品设计太复杂了，用户根本不知道怎么用
助手: 很好的观点！继续说说，你有遇到过什么具体的例子吗？

[... 继续对话，逐步记录洞察 ...]
```

**预期输出**:
- 洞察记录文件（insights.md）
- 写作记录文件（writing-record.md）
- 最终成稿文章

## 📋 PRD 文档撰写助手 (PRD Doc Writer)

### 场景：撰写一个用户登录功能的 PRD

```
用户: 帮我写一个用户登录功能的 PRD
助手: [自动触发 PRD 撰写流程]
助手: 好的！在开始之前，让我们先梳理一下整体的用户旅程。
     这个登录功能主要涉及哪几个阶段？

用户: 主要是注册、登录、找回密码这几个
助手: 明白了。我们梳理出的阶段是：
     1. 用户注册
     2. 用户登录
     3. 找回密码
     这个作为我们后续讨论的'地图'，您看可以吗？

[... 继续逐步讨论每个用户故事 ...]
```

**预期输出**:
- 完整的 PRD 文档
- 包含用户故事、验收标准
- ASCII 线框图展示页面布局
- Mermaid 流程图/状态图

## 🔄 需求变更工作流 (Requirement Change Workflow)

### 场景：修改 Chrome 扩展的侧边栏交互

```
用户: 我想改一下侧边栏的交互，现在点击按钮后直接跳转，我想加个确认弹窗
助手: [自动触发需求变更流程]
助手: 好的，让我们先明确一下这个变更的范围。

     【第1步：澄清变更】
     - 目标：在侧边栏按钮点击后，增加确认弹窗
     - 不改变的：跳转的目标页面、现有数据流
     - 验收标准：
       1. 点击按钮后显示确认弹窗
       2. 点击"确认"后执行原有跳转
       3. 点击"取消"后关闭弹窗

     这样理解对吗？

[... 继续执行基线确认、影响评估、设计提案等步骤 ...]
```

**预期输出**:
- 需求变更简报（change brief）
- 影响评估报告
- 实施后的代码变更
- 回归测试结果
- 决策日志

## 💡 提示

### 自动触发
大多数情况下，你只需要用自然语言描述你的需求，相关的 Skill 会自动触发：

```
❌ 不需要这样：先输入 /thought-mining，然后说"我想写文章"
✅ 直接这样说：我想写一篇关于 XX 的文章
```

### 手动触发
如果自动触发不生效，可以使用斜杠命令：

```
/image-assistant     # 启动配图助手
/thought-mining      # 启动思维挖掘
/prd-doc-writer      # 启动 PRD 撰写
/req-change-workflow # 启动需求变更
```
