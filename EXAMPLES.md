## 🧠 思维挖掘助手 (Thought Mining)

### 场景：想写一篇关于产品设计的文章

```
用户: 我想写一篇关于产品设计的文章
助手: [自动触发思维挖掘流程]
助手: 好的！我们进入思维挖掘阶段。让我们先聊聊，你对产品设计有什么想法？
     随便说，想到什么说什么，我来帮你记录。

用户: 我觉得很多产品设计太复杂了，用户根本不知道怎么用
助手: 很好的观点！继续说说，你有遇到过什么具体的例子吗？

[... 继续对话，逐步记录洞察 ...]
```

**预期输出**:
- 洞察记录文件（insights.md）
- 写作记录文件（writing-record.md）
- 最终成稿文章

## 📋 PRD 文档撰写助手 (PRD Doc Writer)

### 场景：撰写一个用户登录功能的 PRD

```
用户: 帮我写一个用户登录功能的 PRD
助手: [自动触发 PRD 撰写流程]
助手: 好的！在开始之前，让我们先梳理一下整体的用户旅程。
     这个登录功能主要涉及哪几个阶段？

用户: 主要是注册、登录、找回密码这几个
助手: 明白了。我们梳理出的阶段是：
     1. 用户注册
     2. 用户登录
     3. 找回密码
     这个作为我们后续讨论的'地图'，您看可以吗？

[... 继续逐步讨论每个用户故事 ...]
```

**预期输出**:
- 完整的 PRD 文档
- 包含用户故事、验收标准
- ASCII 线框图展示页面布局
- Mermaid 流程图/状态图

## 🔄 需求变更工作流 (Requirement Change Workflow)

### 场景：修改 Chrome 扩展的侧边栏交互

```
用户: 我想改一下侧边栏的交互，现在点击按钮后直接跳转，我想加个确认弹窗
助手: [自动触发需求变更流程]
助手: 好的，让我们先明确一下这个变更的范围。

     【第1步：澄清变更】
     - 目标：在侧边栏按钮点击后，增加确认弹窗
     - 不改变的：跳转的目标页面、现有数据流
     - 验收标准：
       1. 点击按钮后显示确认弹窗
       2. 点击"确认"后执行原有跳转
       3. 点击"取消"后关闭弹窗

     这样理解对吗？

[... 继续执行基线确认、影响评估、设计提案等步骤 ...]
```

**预期输出**:
- 需求变更简报（change brief）
- 影响评估报告
- 实施后的代码变更
- 回归测试结果
- 决策日志

## 💡 提示

### 自动触发
大多数情况下，你只需要用自然语言描述你的需求，相关的 Skill 会自动触发：

```
❌ 不需要这样：先输入 /thought-mining，然后说"我想写文章"
✅ 直接这样说：我想写一篇关于 XX 的文章
```

### 手动触发
如果自动触发不生效，可以使用斜杠命令：

```
/thought-mining      # 启动思维挖掘
/prd-doc-writer      # 启动 PRD 撰写
/req-change-workflow # 启动需求变更
```
