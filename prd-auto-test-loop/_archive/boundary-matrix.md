# 自动化测试边界矩阵

## 适合自动化（建议 AI 覆盖）

- 功能正确性（状态流转、接口处理、校验规则）
- 回归风险路径（历史缺陷复现与防回归）
- 可确定的边界场景（空值、非法值、重复操作）
- 稳定冒烟路径（启动、关键导航、保存/提交成功）
- 可量化非功能指标（超时、错误率、崩溃率）

## 优先人工验证（建议人工兜底）

- 可用性与交互顺滑度
- 视觉质感与品牌一致性
- 业务策略合理性与需求理解偏差
- 开放式探索场景（非常规用户路径）
- 发布优先级与产品取舍判断

## 判定规则

对每条验收标准标记：
- `A`：可完全自动化
- `H`：仅适合人工验证
- `A+H`：自动化 + 人工联合验证
